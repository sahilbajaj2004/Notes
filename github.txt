1. GIT AND GITHUB - INTRODUCTION
================================================================================

GIT:
- Distributed Version Control System (DVCS)
- Tracks changes in source code
- Enables collaboration
- Created by Linus Torvalds (2005)

GITHUB:
- Web-based Git repository hosting service
- Provides collaboration features
- Pull requests, issues, actions
- Largest code hosting platform

VERSION CONTROL BENEFITS:
- Track changes over time
- Collaborate with team members
- Revert to previous versions
- Branch for parallel development
- Merge changes from multiple sources
- Backup and disaster recovery

INSTALLATION:
# Windows: Download from git-scm.com
# Linux
sudo apt-get install git

# macOS
brew install git

# Verify installation
git --version

INITIAL CONFIGURATION:
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
git config --global init.defaultBranch main

# View configuration
git config --list
git config user.name

3. GIT COMMANDS - COMPLETE REFERENCE
================================================================================

REPOSITORY SETUP:
# Initialize new repository
git init

# Clone existing repository
git clone https://github.com/user/repo.git
git clone https://github.com/user/repo.git folder-name
git clone --depth 1 https://github.com/user/repo.git  # Shallow clone

STAGING AND COMMITTING:
# Check status
git status
git status -s           # Short format

# Stage changes
git add file.txt        # Stage specific file
git add .               # Stage all changes
git add *.js            # Stage by pattern
git add -p              # Interactive staging

# Unstage changes
git reset file.txt      # Unstage file
git reset               # Unstage all

# Commit changes
git commit -m "Commit message"
git commit -am "Message" # Stage and commit tracked files
git commit --amend      # Modify last commit

# View history
git log                 # Full log
git log --oneline       # Condensed log
git log --graph         # Graph view
git log -n 5            # Last 5 commits
git log --author="Name" # By author
git log --since="2024-01-01"
git log file.txt        # History of specific file

BRANCHING:
# List branches
git branch              # Local branches
git branch -r           # Remote branches
git branch -a           # All branches

# Create branch
git branch feature      # Create branch
git checkout -b feature # Create and switch
git switch -c feature   # Modern syntax

# Switch branch
git checkout main       # Switch to main
git switch main         # Modern syntax

# Rename branch
git branch -m old-name new-name
git branch -m new-name  # Rename current branch

# Delete branch
git branch -d feature   # Delete merged branch
git branch -D feature   # Force delete

MERGING:
# Merge branch into current
git merge feature

# Merge with commit message
git merge feature -m "Merge feature branch"

# Abort merge
git merge --abort

# Merge strategies
git merge --no-ff feature  # No fast-forward
git merge --squash feature # Squash commits

REBASING:
# Rebase current branch onto main
git rebase main

# Interactive rebase (edit last 3 commits)
git rebase -i HEAD~3

# Abort rebase
git rebase --abort

# Continue after resolving conflicts
git rebase --continue

REMOTE REPOSITORIES:
# View remotes
git remote -v

# Add remote
git remote add origin https://github.com/user/repo.git

# Remove remote
git remote remove origin

# Rename remote
git remote rename origin upstream

# Fetch from remote
git fetch origin
git fetch --all

# Pull from remote
git pull origin main
git pull --rebase origin main

# Push to remote
git push origin main
git push -u origin main    # Set upstream
git push --force           # Force push (careful!)
git push --force-with-lease # Safer force push

STASHING:
# Save changes temporarily
git stash
git stash save "description"
git stash push -m "message"

# List stashes
git stash list

# Apply stash
git stash apply           # Apply latest, keep in stash
git stash pop             # Apply latest, remove from stash
git stash apply stash@{2} # Apply specific stash

# View stash contents
git stash show
git stash show -p         # With diff

# Delete stash
git stash drop stash@{0}
git stash clear           # Delete all

UNDOING CHANGES:
# Discard working directory changes
git checkout -- file.txt
git restore file.txt      # Modern syntax

# Unstage file
git reset HEAD file.txt
git restore --staged file.txt

# Reset to previous commit
git reset --soft HEAD~1   # Keep changes staged
git reset --mixed HEAD~1  # Keep changes unstaged (default)
git reset --hard HEAD~1   # Discard changes

# Revert commit (creates new commit)
git revert HEAD
git revert abc123

VIEWING CHANGES:
# View unstaged changes
git diff

# View staged changes
git diff --staged
git diff --cached

# View changes between commits
git diff abc123 def456

# View changes between branches
git diff main..feature

TAGGING:
# List tags
git tag

# Create lightweight tag
git tag v1.0.0

# Create annotated tag
git tag -a v1.0.0 -m "Version 1.0.0"

# Tag specific commit
git tag v1.0.0 abc123

# Push tags
git push origin v1.0.0
git push origin --tags

# Delete tag
git tag -d v1.0.0
git push origin --delete v1.0.0

CLEANING:
# Remove untracked files (dry run)
git clean -n

# Remove untracked files
git clean -f

# Remove untracked files and directories
git clean -fd

# Remove ignored files too
git clean -fdx

4. GITHUB WORKFLOW
================================================================================

FORK WORKFLOW:
1. Fork repository on GitHub
2. Clone your fork locally
   git clone https://github.com/your-username/repo.git
3. Add upstream remote
   git remote add upstream https://github.com/original/repo.git
4. Create feature branch
   git checkout -b feature
5. Make changes and commit
6. Push to your fork
   git push origin feature
7. Create Pull Request on GitHub
8. Keep fork synced
   git fetch upstream
   git checkout main
   git merge upstream/main

PULL REQUEST WORKFLOW:
1. Create feature branch from main
2. Make changes, commit with meaningful messages
3. Push branch to remote
4. Open Pull Request on GitHub
5. Request reviews
6. Address feedback
7. Merge when approved
8. Delete feature branch

GITHUB CLI (gh):
# Install: https://cli.github.com/

# Authentication
gh auth login

# Repository operations
gh repo create my-repo --public
gh repo clone user/repo
gh repo fork user/repo

# Pull requests
gh pr create --title "Title" --body "Description"
gh pr list
gh pr view 123
gh pr checkout 123
gh pr merge 123

# Issues
gh issue create --title "Bug" --body "Description"
gh issue list
gh issue view 123

6. GITHUB ACTIONS (CI/CD)
================================================================================

AUTOMATED DEPLOYMENT:
# .github/workflows/deploy.yml
name: Deploy to Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Deploy to server
        uses: appleboy/ssh-action@v1.0.0
        with:
          host: ${{ secrets.HOST }}
          username: ${{ secrets.USERNAME }}
          key: ${{ secrets.SSH_KEY }}
          script: |
            cd /var/www/myapp
            git pull origin main
            npm install --production
            pm2 restart all

apt update && apt upgrade -y

cd repo
npm install

# Install PM2 (Process Manager)
sudo npm install -g pm2
pm2 start server.js --name myapp
pm2 startup
pm2 save

NGINX REVERSE PROXY:
# /etc/nginx/sites-available/myapp
server {
    listen 80;
    server_name yourdomain.com www.yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}

# Enable site
sudo ln -s /etc/nginx/sites-available/myapp /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx

12. KEY TECHNOLOGIES OVERVIEW (From All Notes)
================================================================================

HTML:
- HyperText Markup Language - structure of web pages
- HTML5 current standard (Living Standard)
- Semantic elements: header, nav, main, article, section, aside, footer
- Forms: input types, validation, accessibility
- Media: audio, video, iframe
- SEO: meta tags, Open Graph, Twitter Cards

CSS:
- Cascading Style Sheets - styling and layout
- Box Model: content, padding, border, margin
- Flexbox: one-dimensional layout (justify-content, align-items)
- Grid: two-dimensional layout (grid-template-columns, grid-template-rows)
- Positioning: static, relative, absolute, fixed, sticky
- Responsive: media queries, viewport units, clamp()
- Animations: transitions, keyframes, transforms
- Modern: CSS Variables, container queries, :has() selector

JavaScript:
- High-level, interpreted programming language
- ES6+ features: let/const, arrow functions, destructuring, modules
- Data types: string, number, boolean, null, undefined, symbol, bigint
- Arrays: map, filter, reduce, find, some, every
- Objects: destructuring, spread, Object methods
- Async: Promises, async/await, fetch API
- DOM manipulation, event handling
- Closures, higher-order functions, prototypes

Node.js:
- JavaScript runtime built on Chrome's V8 engine
- Non-blocking, event-driven architecture
- Core modules: fs, path, http, os, events, crypto
- NPM package manager
- Streams and Buffers for data handling
- Environment variables with dotenv

Express.js:
- Fast, minimalist web framework for Node.js
- Routing: GET, POST, PUT, PATCH, DELETE
- Middleware: application-level, router-level, error-handling
- Request/Response objects
- Template engines: EJS, Pug, Handlebars
- Static file serving
- RESTful API design with proper status codes
- Authentication with JWT

MongoDB:
- NoSQL document-oriented database
- CRUD operations: insertOne, find, updateOne, deleteOne
- Query operators: $eq, $gt, $lt, $in, $and, $or, $regex
- Update operators: $set, $unset, $inc, $push, $pull
- Aggregation pipeline: $match, $group, $sort, $lookup, $unwind
- Indexes: single, compound, text, TTL, unique
- Mongoose ODM: schemas, models, virtuals, middleware, populate

React.js:
- JavaScript library for building UIs (by Meta)
- JSX syntax, Components (functional), Props
- Hooks: useState, useEffect, useRef, useContext, useReducer

Next.js:
